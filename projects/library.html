Overview

앱의 정보 구조를 재설계해,
사용자가 '지금 할 수 있는 행동'을 바로 이해하도록

기존의 서비스는 정보는 충분하지만, 정보 구조와 안내가 부족해 사용자가 '지금 나에게 중요한 정보'와 '다음 행동'을 빠르게 찾지 못했습니다.

현재 상태 인지 → 다음 행동 제안 → 기록과 성취를 보여주는 구조로 재설계. 찾아보는 서비스에서, 알아서 안내하는 서비스로 전환했습니다.

Research

정보가 없는 것이 아니라,
지금 필요한 정보가 보이지 않는 서비스였습니다

리뷰 분석과 사용자 6명 대상 설문조사를 통해 문제를 파악했습니다. 앱 가이드가 충분하지 않다 85%, 대출 과정이 명확하지 않다 70%의 응답을 얻었습니다.

리뷰 분석 — 주요 단점

앱 강제종료 및 느린 로딩, 비일관적인 내비게이션, 불편한 로그인 방식, 글자 크기 커스터마이징 제약

설문 인사이트

정보 구조와 안내 방식 개선 필요성 확인. 기술·정책·UX가 각각 따로 움직이며 사용자가 불일치를 떠안고 있음

찾아보는 → 알아서 안내하는

회원/비회원 서비스 가치가 명확히 구분되지 않아 첫 방문자가 가입 이유를 충분히 설득받지 못함
→
회원가입 전후 차별화된 가치를 명확히 보여주고, 첫 방문자가 바로 시작할 수 있도록 안내
로그인 이후에도 현재 상태와 가능한 행동이 보이지 않아 매번 깊은 메뉴까지 반복 탐색
→
대출·예약 현황과 반납일 등 핵심 정보를 첫 화면에서 바로 확인하는 개인화 대시보드 제공
개인화 추천·신규 도서·반납 임박 알림이 홈에서 자연스럽게 드러나지 않아 재방문 동기 약함
→
매번 방문마다 새 콘텐츠와 맞춤 추천, 반납 임박 정보를 자동으로 노출해 재방문 유도

Design Process

리서치

리뷰 분석 · 설문조사 — 구글·앱스토어 리뷰 분석 및 사용자 6명 설문. 정보 구조와 안내 방식 개선 필요성 확인. (60%)

IA 설계

상태 중심 IA 재구성 — '현재 상태 인지 → 다음 행동 제안 → 기록·성취 축적' 흐름으로 탭별 역할을 탐색·관리·동기부여로 분리. (60%)

디자인 시스템

경기 파랑색 기반 시스템 — 신뢰와 미래가치를 뜻하는 경기 파랑색(#09408A)·보조색(#0AA4B4) 사용. 프리텐다드 폰트, 컴포넌트 라이브러리 구축.

핵심 화면 설계

홈 · 검색 · 내 서재 · 마이페이지 — 모바일 환경 최적화 컴포넌트. 대출·예약·완료 상태별 카드 설계. (80%)

UT 기반 개선

검색 필터 UI 개선 — 실제 사용자 테스트 결과를 바탕으로 검색 필터 UI를 개선하여 검색 성공률 및 속도 향상.

기능 나열 중심 UI에서 사용자 상태·다음 행동 중심 UI로 전환하는 설계 경험을 쌓을 수 있었습니다. IA 재구성과 컴포넌트 시스템 설계를 통해 확장 가능한 구조를 만들고자 했습니다.
